# Project Overview

This repository documents the end-to-end process of securely configuring an AWS account from scratch.

Rather than simply creating cloud resources, this project focuses on implementing security, governance, identity management, billing controls, and cost monitoring using AWS best practices.

The documentation serves as both a personal engineering portfolio and a repeatable deployment guide for anyone wishing to build a secure AWS foundation.

---

# Objectives

The goals of this project include:

- Secure the AWS Root Account
- Configure Multi-Factor Authentication (MFA)
- Create an IAM Administrator Account
- Implement Least Privilege principles
- Enable IAM Billing Access
- Configure Billing Permissions
- Monitor Promotional Credits
- Create AWS Budgets
- Configure Cost Monitoring
- Document every implementation step
- Produce professional cloud documentation

# Security Implementations

This project implements the following security controls:

- Root account protected with MFA
- Dedicated IAM Administrator created
- Administrator group using AWS managed policies
- Billing access delegated to IAM
- Cost monitoring enabled
- Budget alerts configured
- Promotional credits monitored
- Separation of Root and IAM administrative access
