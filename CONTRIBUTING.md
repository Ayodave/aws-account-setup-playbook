# Contributing

Thank you for your interest in this project.

Although this repository currently serves as a personal cloud engineering portfolio, contributions, suggestions, and improvements are welcome.

---

## Reporting Issues

Please open a GitHub Issue if you discover:

- Documentation errors
- Incorrect AWS procedures
- Security concerns
- Broken links
- Typographical mistakes

---

## Pull Requests

Contributors should ensure that:

- Documentation follows Markdown best practices.
- AWS terminology is accurate.
- Security recommendations align with AWS Well-Architected guidance.
- Screenshots do not expose sensitive information.

---

## Documentation Standards

Every implementation guide should include:

- Executive Summary
- Objectives
- Business Context
- Implementation Overview
- Validation
- Lessons Learned
